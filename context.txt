# Контекст разработки проекта «ЛитКот» (LeetCat)

Мяу! Этот файл содержит полный слепок состояния проекта, зафиксированный на момент переноса разработки в директорию `C:\source\ЛитКот`. Используйте его для восстановления контекста ИИ-ассистента в любой новой сессии.

## 📝 Основная информация
* **Название проекта:** ЛитКот (бывший ЛитКот)
* **Разработчик:** Arlist Tech (логотип: `https://arlist.tech/icon.svg`)
* **Фирменный стиль:** LeetCode-style, принудительная темная тема (`bg-zinc-950`), контрастные розово-фиолетовые акценты (`#d946ef`), основной шрифт для брендинга — **Dela Gothic One**.
* **Главный талисман:** ASCII-котики везде (в шапке, на главной, в фавиконке и внутри задач).

## 🛠 Выполненные технические изменения

### 1. Архитектура и Docker (Режим Production)
* Настроен многоэтапный Docker-манифест (`Dockerfile`) для сборки проекта в режиме **Production** с предварительной компиляцией бандла, что исключает задержки динамической компиляции.
* В образ добавлены `openssl` (необходим для Prisma Engine) и глобальный пакет `dotenv-cli`.
* Оптимизирована команда `pnpm install` (убран флаг `--frozen-lockfile` для корректного обновления зависимостей прямо во время сборки).
* Настроен `docker-compose.yaml` с тремя сервисами: `db` (MySQL), `redis` (порт перевешен на `6380` во избежание конфликтов) и `app` (основное Next.js приложение).

### 2. Кастомная авторизация и Почтовая готовность
* Схема базы данных `schema.prisma` расширена: в модель `User` добавлено опциональное поле `password` для автономной регистрации.
* В проект интегрирована библиотека `bcryptjs` для надежного хеширования паролей пользователей.
* Создана полноценная страница регистрации `/register` с API-роутом `/api/register` и страница входа `/login`.
* Авторизация переведена на `CredentialsProvider` от `NextAuth` с использованием JWT-стратегии сессий.
* **Милая фича:** После успешной регистрации котика встречает кастомное сообщение: *«Привет! Я ЛитКот. Мы очень мяв рады видеть тебя в нашей банде! Твой профиль уже прогрет и готов к приключениям. Давай теперь войдем?»*

### 3. Ребрендинг и Интерфейс
* Старый бренд "ЛитКот BETA" полностью вырезан. В шапку добавлен текстовый логотип «ЛитКот БЕТА» шрифтом *Dela Gothic One* и мини-ASCII кот `/\_/\ ( o.o )`.
* Главный логотип на `hero.tsx` заменен на полноценный pre-блок с котиком.
* Из папки `apps/web/public` удален весь старый графический мусор (`santa-hat.png`, `avatar.jpeg` и т.д.).
* Скачан и подключен официальный логотип `arlist-logo.svg` от Arlist Tech.
* Полностью переписан подвал (`footsies.tsx`): удалены все ссылки на внешние репозитории оригинального ЛитКот, добавлена надпись «Сделано с любовью Arlist Tech» и актуальный копирайт.
* Изменен файл `favicon.svg` — теперь там отображается фирменный фиолетовый ASCII-кот.

## 📚 Рехоул курсов и «Умные» кошачьи задачи

Архитектура базы данных дополнена перечислением `Language` (`TYPESCRIPT`, `JAVASCRIPT`, `PYTHON`, `GO`) для полноценной мультиязычной платформы. Созданы и полностью прописаны (без заглушек!) первые глубокие задачи:

1. **TypeScript (Начальный уровень):** *«Примитивные типы (Основы ЛитКота)»*
   * **Сюжет:** Заполнение медицинского паспорта кота Барсика (типизация имени, возраста и флага счастья), а также типизация функции его кормления `feedCat(name: string, weight: number)`.
2. **TypeScript (Средний уровень):** *«Утилита Выбора (Pick)»*
   * **Сюжет:** Создание уменьшенной копии кошачьего профиля из огромного интерфейса характеристик приюта с помощью кастомного Mapped Type `MyPick<T, K extends keyof T>`.
3. **Python (Начальный уровень):** *«Кошачий Python: Первое Мяу»*
   * **Сюжет:** Написание алгоритма приветствия стаи котов с использованием списковых включений (*List Comprehensions*) и циклов `for`.
4. **Go (Средний уровень):** *«Быстрый как Кот: Введение в Go»*
   * **Сюжет:** Проектирование кошачьих структур (`struct`) и реализация методов приемников для генерации быстрого и эффективного «Мяу».
